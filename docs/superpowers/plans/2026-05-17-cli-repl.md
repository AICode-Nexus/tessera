# CLI REPL Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a usable interactive `tessera chat` REPL that feels like a lightweight Claude Code / Codex CLI chat shell.

**Architecture:** Keep the REPL in `crates/cli`; use `tessera-client::ClientSnapshot` for state and route all model work through `run_chat_with_config_and_events`. The REPL owns stdio and command parsing only.

**Tech Stack:** Rust, Clap, Tokio, existing Tessera core/client/provider/storage crates.
