# Tessera v0.1 Release Checklist

日期：2026-05-15

本文定义 v0.1 的发布门禁、候选范围、tag 策略和回滚方式。它不替代 [Global Plan](global-plan.md)，而是把“什么时候可以打 tag”说清楚。

## 1. Release Goal

v0.1 的目标不是完整产品，而是一个质量优先、可回放、可审计、AI-friendly 的最小本地运行体：

- Rust workspace 和 crate 边界稳定。
- CLI、TUI、未来 GUI 共享同一套 headless runtime 方向。
- mock provider 可以驱动完整 chat run。
- core 会把 provider stream 转成 `EventFrame` 并写 trace。
- JSONL 是事件真相，SQLite 是可重建索引。
- TUI 可以基于 shared `tessera-client` model 完成最小聊天、profile switch、`/new`、`/save`、`/export`。
- replay fixture 可以在没有真实 provider 的情况下验证核心行为。

## 2. Version Recommendation

当前建议分两步：

| Version | 推荐状态 | 理由 |
| --- | --- | --- |
| `v0.1.0-alpha.1` | 已打 tag | 离线 mock、trace、replay、CLI、TUI、client model 已形成闭环 |
| `v0.1.0` | 暂缓 | 真实 provider smoke 尚未在当前环境完成 |

如果要更保守，先只打 `v0.1.0-alpha.1`，把正式 `v0.1.0` 留给真实 OpenAI-compatible 或 Ollama smoke 通过之后。

## 3. Release Scope

### Included

- `protocol`：typed IDs、runtime objects、`RunEvent`、`EventFrame`、provider capability、usage/cache/cost/route decision schema。
- `client`：UI-neutral `ClientIntent`、`ClientStatus`、`ClientProjection`、`ClientSnapshot`。
- `core`：conversation engine、event routing、trace persistence coordination、cancellation/timeout/backpressure basics、replay runner。
- `providers`：mock provider、OpenAI-compatible adapter、Ollama adapter、parser tests、opt-in live smoke tests。
- `storage`：JSONL trace writer、SQLite index、runtime object queries、index rebuild。
- `config`：TOML provider profile loading、env-only secret reference model。
- `cli`：`doctor --json`、mock/config-routed `chat`、TUI command dispatch.
- `tui`：Ratatui chat shell、profile switch、live event rendering、bounded live channel、`/new`、`/save`、`/export` basics。
- Documentation：architecture, crate boundaries, protocol, trace schema, GUI-ready ADR, DeepSeek-TUI lessons, global plan.

### Explicitly Excluded

- Tool execution.
- Shell/file/git tools.
- MCP runtime.
- Agent runtime.
- Swarm scheduler.
- Long-term memory runtime.
- Learning runtime.
- Auto router.
- GUI implementation.
- Complex multi-window TUI.

## 4. Tag Blockers

### Blockers for `v0.1.0-alpha.1`

- [x] Local required gates pass:
  - `PATH="$HOME/.cargo/bin:$PATH" cargo fmt --all -- --check`
  - `PATH="$HOME/.cargo/bin:$PATH" cargo clippy --workspace --all-targets -- -D warnings`
  - `PATH="$HOME/.cargo/bin:$PATH" cargo test --workspace`
  - `git diff --check`
- [x] Smoke commands pass:
  - `PATH="$HOME/.cargo/bin:$PATH" cargo run -p tessera-cli -- doctor --json`
  - `PATH="$HOME/.cargo/bin:$PATH" cargo run -p tessera-cli -- chat --provider mock --prompt "hello"`
  - `PATH="$HOME/.cargo/bin:$PATH" cargo run -p tessera-cli -- tui --help`
- [x] CI is green on `main`.
- [x] `CHANGELOG.md` has a `v0.1.0-alpha.1` section or release note draft.
- [x] Working tree is clean before tagging.
- [x] No secret material is present in trace fixtures, docs, tests, or logs.

### Blockers for final `v0.1.0`

- [x] All `v0.1.0-alpha.1` blockers are cleared.
- [ ] At least one real provider smoke path is verified:
  - OpenAI-compatible with `TESSERA_OPENAI_COMPATIBLE_BASE_URL` and related env vars, or
  - Ollama with reachable `localhost:11434` and a local model.
- [x] Provider error normalization is implemented for provider HTTP errors, provider parse errors, retryability, safe details, and pre-trace redaction.
- [ ] Real provider success path is checked for secret redaction in debug/log/trace surfaces.
- [ ] Release notes state known limitations clearly.

## 5. Pre-Tag Commands

Run from repo root:

```bash
git pull --ff-only origin main
git status --short --branch
PATH="$HOME/.cargo/bin:$PATH" cargo fmt --all -- --check
PATH="$HOME/.cargo/bin:$PATH" cargo clippy --workspace --all-targets -- -D warnings
PATH="$HOME/.cargo/bin:$PATH" cargo test --workspace
git diff --check
PATH="$HOME/.cargo/bin:$PATH" cargo run -p tessera-cli -- doctor --json
PATH="$HOME/.cargo/bin:$PATH" cargo run -p tessera-cli -- chat --provider mock --prompt "hello"
PATH="$HOME/.cargo/bin:$PATH" cargo run -p tessera-cli -- tui --help
```

Optional live provider gates:

```bash
PATH="$HOME/.cargo/bin:$PATH" cargo test -p tessera-providers --test live_smoke -- --ignored
```

Only run the ignored live smoke tests when the required provider env vars or local services are present.

## 6. Tag Plan

### Alpha tag

```bash
git tag -a v0.1.0-alpha.1 -m "Tessera v0.1.0-alpha.1"
git push origin v0.1.0-alpha.1
```

Use this tag for the first public checkpoint of the offline closed loop.

### Final tag

```bash
git tag -a v0.1.0 -m "Tessera v0.1.0"
git push origin v0.1.0
```

Only use this after the final blockers are cleared.

## 7. Release Notes Draft

### Highlights

- Rust-first headless runtime with provider-neutral events.
- JSONL trace as event truth plus rebuildable SQLite index.
- Mock-driven offline chat loop with replay fixture.
- OpenAI-compatible and Ollama provider adapter skeletons with parser tests and opt-in live smokes.
- Shared UI-neutral client model for TUI and future GUI.
- Minimal Ratatui TUI with profile switching, live event rendering, bounded backpressure, `/new`, `/save`, and `/export`.
- GUI direction documented through ADR-001: Tauri-first product GUI, no GUI runtime fork.

### Known Limitations

- Real provider smoke has not necessarily been verified in every checkout.
- `/save` and `/export` are v0.1 basics over client projection, not full file export workflow.
- Real provider success-path redaction still needs live smoke evidence.
- No tool execution, agent runtime, MCP runtime, long-term memory runtime, Auto router, or GUI implementation.

## 8. Rollback

If a tag is pushed incorrectly:

```bash
git tag -d v0.1.0-alpha.1
git push origin :refs/tags/v0.1.0-alpha.1
```

Only delete a public tag if it is clearly wrong and the project owner agrees. Prefer issuing a follow-up `alpha.2` tag if the tag was merely incomplete.

## 9. Post-Tag Follow-Up

- Create GitHub release notes from the release notes draft.
- Link the tag from `README.md`.
- Move unfinished items into the next section of `docs/global-plan.md`.
- Start the next implementation slice from real provider smoke verification.
